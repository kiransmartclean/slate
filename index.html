    Introduction

    Welcome to the Smartclean API docs! The documentations for multiple modules can be accesed here. You can view the sample request and code examples in the dark area to the right.

    Login

    Login: To authorize and use the further requests, please use this code:

    curl --location --request POST 'https://services.smartclean.io/api/scteams/scdashboard/unauth/v3/service/:serviceid/login' \
--header 'Content-Type: application/json' \
--data-raw '{
    "uname":  "<USER_NAME>", 
    "passwd": "<PASSWORD>"
}'

    Make sure to replace <USER_NAME> and <PASSWORD> with your username and password.

    This endpoint logs a user into the system. All the further API requests uses a token to get access and expects the token to be included in a header that looks like the following:

    Authorization: <TOKEN>

    You must replace TOKEN with the ID token from the response.

    HTTP Request

    POST https://services.smartclean.io/api/scteams/scdashboard/unauth/v3/service/:serviceid/login

    Path Parameters

    Parameter Default Description
    serviceid empty Provide the caller's service name

    Raw Body

    { "uname": "USER_NAME", "passwd": "PASSWORD" }

    You must replace USER_NAME and PASSWORD with your username and password.
    On success this returns a list of tokens, from which the ID token is used for furthier requests

    Zones

    Create a new Zone

    Create a new Zone

    curl --location --request POST 'https://services.smartclean.io/api/scteams/scdashboard/auth/v3/service/:serviceid/projects/:pid/zones' \
--header 'Authorization: <ID_TOKEN>' \
--data-raw '{
  "name": "<NAME>"
}'

    Make sure to replace <NAME> with name to the zone and <ID_TOKEN> with the ID token that you got from login endpoint

    The above command returns JSON structured like this:

    {
    "status": "Success",
    "message": "Created the zone <ZoneID>",
    "data": {
        "ATTR": "attr#zones#details#<ProjectID>#<ZoneID>",
        "CreatedOn": 1598347752,
        "UpdatedOn": 1598347752,
        "Data": {
            "Name": "<NAME>"
        },
        "ID": "SCProjects",
        "NS": "PROJECT_ZONE_DETAILS",
        "SUB1": "<ZoneID>",
        "ZoneId": "<ZoneID>",
        "Tags": []
    }
}

    This endpoint creates a new zone with a given name

    HTTP Request

    POST https://services.smartclean.io/api/scteams/scdashboard/auth/v3/service/:serviceid/projects/:pid/zones

    Path Parameters

    Parameter Default Description
    serviceid empty Provide the caller's service name
    pid empty Provide a project ID against which the zone has to be creates

    Raw Body

    { "name": "NAME" }

    You must replace NAME some name to the new zone.
    On success the API creates a new zone

    Errors

    This error section is stored in a separate file in includes/_errors.md. Slate allows you to optionally separate out your docs into many files...just save them to the includes folder and add them to the top of your index.md's frontmatter. Files are included in the order listed.

    The Kittn API uses the following error codes:

    Error Code Meaning
    400 Bad Request -- Your request is invalid.
    401 Unauthorized -- Your API key is wrong.
    403 Forbidden -- The kitten requested is hidden for administrators only.
    404 Not Found -- The specified kitten could not be found.
    405 Method Not Allowed -- You tried to access a kitten with an invalid method.
    406 Not Acceptable -- You requested a format that isn't json.
    410 Gone -- The kitten requested has been removed from our servers.
    418 I'm a teapot.
    429 Too Many Requests -- You're requesting too many kittens! Slow down!
    500 Internal Server Error -- We had a problem with our server. Try again later.
    503 Service Unavailable -- We're temporarily offline for maintenance. Please try again later.
